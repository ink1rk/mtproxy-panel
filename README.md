# MTProxy Control Panel

Веб-панель управления Telegram MTProxy на FastAPI + Docker SDK.
Позволяет создавать, просматривать и удалять MTProxy-контейнеры
через браузер, без ручного редактирования конфигов.

## Возможности

- Аутентификация по логину/паролю (сессионные куки, пароль хешируется PBKDF2-HMAC-SHA256 с солью)
- Современный тёмный интерфейс: карточки-статистика, поиск и сортировка таблицы,
  модальные окна, копирование в буфер обмена, автообновление статусов без перезагрузки страницы
- Создание MTProxy кнопкой (официальный образ `telegrammessenger/proxy`)
- **Выбор порта**: случайный свободный или указанный вручную (с проверкой занятости)
- **Режимы обфускации secret**:
  - `classic` — обычный 32-символьный hex-секрет
  - `dd` — random padding, затрудняет обнаружение через DPI
  - `ee` — fake-TLS: трафик маскируется под HTTPS-соединение к выбранному домену
- Список всех прокси: статус контейнера, IP, порт, secret, режим, дата создания
- Ссылки `tg://proxy?...` и `https://t.me/proxy?...`, QR-код для быстрого подключения
- Удаление с подтверждением (контейнер, QR и запись в БД удаляются полностью)
- Полный откат при любой ошибке создания — "полуготовых" прокси не остаётся

## Стек

Python 3.12, FastAPI, Jinja2, SQLite, Docker SDK, Bootstrap 5, qrcode, Pillow, uvicorn.
Без docker-compose, systemd, redis, celery, nginx, ssl-терминации и JS-фреймворков
(интерактивность — на чистом vanilla JS).

## Архитектура

```
main.py             # точка входа FastAPI, логирование, lifespan, SessionMiddleware
routes.py            # HTTP-роуты (только вызовы service.py / auth.py)
service.py           # сервисный слой: бизнес-логика создания/удаления прокси
mtproxy.py           # оркестрация провижининга контейнера + rollback
docker_manager.py    # обёртка над Docker SDK
repository.py        # Repository Pattern поверх SQLite (proxies)
auth.py              # аутентификация: хеширование паролей, сессии, repository админов
database.py          # соединение с SQLite, создание таблиц, авто-миграция схемы
models.py            # доменные модели Proxy / AdminUser
schemas.py           # Pydantic v2 схемы
utils.py             # порты, secret (classic/dd/ee), QR, публичный IP
config.py            # все настройки в одном месте
templates/           # base.html, login.html, index.html
static/qr/           # сгенерированные QR-коды
logs/                 # логи с ротацией (RotatingFileHandler)
```

## Установка на Ubuntu Server

```bash
git clone <URL_ТВОЕГО_РЕПОЗИТОРИЯ>.git
cd mtproxy-panel
bash install.sh
```

Скрипт автоматически:

1. установит Python 3 (и модуль `venv`/`pip`, даже если Python уже был) и Docker, если их нет;
2. создаст `venv` и поставит зависимости;
3. создаст структуру каталогов и SQLite-базу;
4. запустит FastAPI в фоне (`nohup ... &`, без systemd);
5. подождёт запуск и проверит `curl -L http://127.0.0.1:8000/` -> HTTP 200 + HTML
   (панель теперь требует логин, поэтому проверка идёт с учётом редиректа на `/login`);
6. проверит `docker ps`;
7. **выведет логин и пароль первой учётной записи администратора** (только один раз —
   пароль хранится в базе только в виде хеша и повторно нигде не показывается);
8. если что-то не так — завершится с ошибкой и покажет последние строки лога.

После успешной установки открой в браузере:

```
http://<IP_СЕРВЕРА>:8000/
```

и войди под логином/паролем, показанным в конце установки. Пароль можно сменить
в самой панели (кнопка «Пароль» в правом верхнем углу).

### Остановка / перезапуск

```bash
kill $(cat run.pid)      # остановить
bash install.sh          # перезапустить (сам остановит старый процесс)
```

При повторном запуске (например, после `git pull` или для простого перезапуска)
`install.sh` сам подтягивает последний коммит из `origin` для текущей ветки
(`git fetch` + `git reset --hard`, только если в рабочей копии нет несохранённых
изменений) и очищает кеш `__pycache__` — это гарантирует, что роуты (`routes.py`)
и шаблоны (`templates/`) всегда берутся из одного и того же коммита и не
рассинхронизируются между перезапусками.

### Логи

```bash
tail -f logs/uvicorn.log   # логи процесса uvicorn
tail -f logs/app.log       # логи приложения (RotatingFileHandler)
```

## Режимы secret подробнее

При создании прокси в модальном окне можно выбрать:

- **classic** — обычный секрет, наиболее совместим, но легче всего детектируется DPI.
- **dd** — тот же базовый секрет, но в ссылке добавляется префикс `dd`, включающий
  случайный паддинг пакетов на стороне клиента. Дополнительных настроек сервера не требует.
- **ee** — fake-TLS. Указывается домен (например, `www.google.com`), под HTTPS-соединение
  к которому будет маскироваться трафик. Секрет в этом случае имеет вид
  `ee` + 32 hex-символа + hex-encoded домен, и именно он передаётся в контейнер как `SECRET`.

Все проверки создания (запуск контейнера, открытие порта, ответ на подключение) выполняются
одинаково для всех режимов — если `ee`-секрет с указанным доменом не заработает с конкретной
версией образа, панель автоматически откатит контейнер и покажет ошибку, не оставляя
"подвисших" прокси.

## Примечания по безопасности

- Base-secret генерируется через `secrets.token_hex(16)`, не `random`, не `uuid`.
- Пароль администратора хранится только в виде PBKDF2-HMAC-SHA256 хеша с уникальной солью
  (260 000 итераций), сам пароль нигде не сохраняется в открытом виде.
- Сессионная кука подписывается ключом, который генерируется один раз при первом запуске
  и хранится в `data/session_secret.key` (исключён из `.gitignore`).
- Все входные данные, отображаемые в HTML, экранируются (`html.escape` / автоэкранирование Jinja2).
- Никаких `os.system` — только `subprocess`/Docker SDK с явными таймаутами.
- Панель по умолчанию слушает `0.0.0.0:8000` без TLS. Для доступа из интернета рекомендуется
  закрыть порт 8000 файрволом и открывать доступ, например, через SSH-туннель, либо
  добавить TLS самостоятельно на внешнем уровне.
